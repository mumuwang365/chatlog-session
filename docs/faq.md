# 常见问题 (FAQ)

这里列出了用户在使用 Chatlog Session 时可能遇到的常见问题。

## 基础常见问题

### Q: 为什么我的图片无法显示？

1. 确保您的 Chatlog API 服务正常运行。
2. 检查 **设置** 中的 API 地址是否正确。
3. 由于 chatlog 需要 img_key 才能访问图片资源，请确保您的 API 服务已正确配置并能够提供 img_key。
4. 如果是在线版（HTTPS）连接本地 HTTP API，可能会遇到 Mixed Content 问题。建议使用 Chrome 浏览器并允许不安全内容，或将 API 部署为 HTTPS。

### Q: 数据是存在哪里的？

所有数据仅存储在您的浏览器 **本地缓存** (IndexedDB) 中。当您清除浏览器缓存或卸载 PWA 应用时，这些数据会被清除。我们不会上传任何数据到云端。

## 进阶与功能问题

### Q: 请问支持 4.x 还是 3.x 版本的微信？

**Chatlog Session** 是一个前端展示工具，它依赖于后端的 [Chatlog](https://github.com/sjzar/chatlog) API 来读取数据。因此，具体的微信版本支持情况完全取决于后端 `chatlog` 项目的支持能力。

请参考 `chatlog` 项目的文档或发布说明以确认其当前支持的微信版本。

### Q: 为什么视频无法播放或显示缺失？

**问题现象**：
在 Chatlog Session 中查看视频时无法播放，或者部分图片/视频显示异常。这通常发生在 PC 微信客户端未自动下载该视频（例如视频大小超过自动下载限制）的情况下。

**原因**：
Chatlog Session 的原理是读取您本地磁盘上已有的微信数据文件。如果 PC 微信客户端本身没有下载该视频或图片文件，那么本地磁盘上就不存在该文件数据。既然文件不存在，Chatlog Session 自然无法读取和展示。

**解决方案**：
这是由于微信客户端的“懒加载”机制导致的，属于数据源缺失。
要解决此问题，您需要：
1. 在 PC 微信设置中，检查并调整文件/视频的自动下载设置（例如调大自动下载的大小限制）。
2. 对于视频或图片，需要在 PC 微信客户端中手动点击查看或播放，确保文件已成功下载到本地磁盘。
